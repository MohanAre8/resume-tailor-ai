1. System Overview

Purpose

Resume Tailor is an intelligent resume optimization system that helps job seekers customize their resumes for specific positions. Using a multi-agent AI architecture, the system:

  • Research company-specific context and tech stacks via web search
  • Score original resume against job requirements
  • Intelligently improve resume with metrics-focused, concise bullets
  • Iteratively refine until optimal ATS score achieved (target: 85%+)
  • Track applications in Google Sheets automatically

Key Features

  • ✅ Multi-format Resume Support: PDF, DOCX, Google Docs, plain text
  • ✅ Company Research Integration: Web search via Tavily API for real tech stack context
  • ✅ Original Resume Scoring: Baseline ATS score before improvement
  • ✅ Concise, Metric-Driven Bullets: Optimized for recruiter 6-second scan
  • ✅ Iterative Refinement: Max 2 iterations with progressive scoring
  • ✅ Complete Resume Output: No truncation, all sections preserved
  • ✅ Google Drive Integration: Auto-upload tailored resume
  • ✅ Application Tracking: Auto-log to Google Sheets
  • ✅ Privacy-First Design: Uses user's own Google Drive/Sheets

Processing Pipeline

Pipeline Steps:
  1. Parse Resume: Extract text from PDF/DOCX (unpdf, mammoth)
  2. Research Agent: Web search company context (Tavily API) → Keywords
  3. Score Agent: Baseline score on original resume
  4. Tailor Agent: Improve resume with metrics & keywords
  5. Critic Agent: Score improved resume, provide feedback
  6. Loop Router: If score < 90 AND iterations < 2, loop to step 4
  7. Upload: Save tailored resume to Google Drive, log to Sheets

4. Multi-Agent System Design

Agent 1: Researcher (Web Search)

🔍 Purpose

Extract real company context and tech stack via web search

Inputs

  • Company name
  • Job role/title

Outputs

  • 40+ key technologies used by company
  • Company insights (projects, focus areas)

Model Configuration

Llama 3.1 8B (Groq) Fast Low Cost

Agent 2: Tailor (Resume Improvement)

✏️ Purpose

Rewrite resume bullets to be concise, metric-driven, and keyword-optimized

Key Characteristics

  • Concise Bullets: Max 1-2 lines per bullet point
  • Metric-First: Lead with %/$ outcomes
  • Verb-Rich: Architected, Optimized, Accelerated, not "Worked on"
  • Complete Output: All sections, all content preserved
  • Format: "Did [action] → [quantifiable result]"

Inputs

  • Current resume
  • Job description
  • Keywords from research agent
  • Feedback from previous iteration (if loop)

Outputs

  • Complete improved resume (same format, better content)

Model Configuration

Llama 3.1 8B (Groq) 3500 maxTokens Room for full resume

Agent 3: Critic (ATS Scoring)

🎯 Purpose

Score resume on ATS match & recruiter appeal. Drive iterative improvement.

Scoring Rubric (0-100)

  • Quantifiable Metrics (35%): % improvements, $savings, numbers in every bullet?
  • Keyword Match (35%): Job terms + technical stack coverage?
  • Conciseness & Scannability (20%): Recruiter can scan in 6s? Power verbs?
  • Structure (10%): All sections present, formatting clean?

Bonus Points For

  • "Increased X by 50%", "Reduced latency by 40%", "$2M project delivered"
  • Natural keyword integration
  • Tight, punchy language

Model Configuration

Llama 3.3 70B (Groq) Better Reasoning More Nuanced Scoring

6. Scoring Algorithm & Optimization

Scoring Methodology

The Critic agent scores resumes (0-100) using a weighted rubric designed to reward what actually matters to recruiters:

Dimension Weight What We Score High Score Indicators
Quantifiable Metrics 35% Does every bullet have measurable outcome? "Increased x by 50%", "$2M saved", "40% faster"
Keyword Match 35% Job terms + tech stack coverage Key terms from JD appear naturally in resume
Conciseness 20% Scannable in 6 seconds? Power verbs? Short bullets, "Architected" not "Worked on"
Structure 10% All sections present? Proper formatting? Contact, Summary, Experience, Skills, Education

Improvement Example

Original Bullet

"Worked on improving the performance of the web application"

Improved Bullet (Post-Tailor)

"Optimized React app reducing page load by 45% → 0.8s response time"

Score Impact

  • Metrics: +25 points (specific %, time saved)
  • Conciseness: +20 points (1 line, power verb "Optimized")
  • Overall: 72% → 87% (15 point gain)

Score Interpretation

90-100: EXCELLENT
Ready to submit. All boxes checked - metrics, keywords, concise, structured.
80-89: GOOD
Competitive. Minor improvements possible. May loop for refinement.
70-79: FAIR
Improvement needed. Missing keywords or metrics. Should loop.
<70: NEEDS WORK
Significant gaps. May need manual revision beyond AI.

7. Optimizations & Performance

Token & Cost Optimization

Designed to minimize token usage while maintaining quality:

  • Concise Prompts: Tailor prompt ~400 tokens (vs. 1200+ verbose)
  • Keyword Extraction: No AI used. Pure regex matching (~0 tokens)
  • Output Tokens: Resume output ~1500 tokens. Max 3500 allocated to prevent truncation
  • Max 2 Iterations: Caps total processing at ~8K tokens per job
  • No Logging: Resume content not logged to prevent API rate issues

8. Security & Privacy

Privacy-First Architecture

Data Storage

  • ✅ User's own Google Drive: Tailored resume stored in user's account
  • ✅ User's own Google Sheets: Application tracking stored in user's account
  • ✅ No central database: No user data persisted on our servers
  • ✅ Session-only state: Temporary data discarded after processing

Authentication & Authorization

  • NextAuth: Secure OAuth 2.0 flow with Google
  • Scopes Limited: Only request Drive, Sheets permissions (not Gmail, etc.)
  • Token Refresh: Automatic token management with secure refresh logic
  • Session Management: Secure cookies with HttpOnly, SameSite flags

Data in Transit

  • HTTPS Only: All API calls encrypted end-to-end
  • No Resume Logging: Resume content not logged in production
  • Secure Google APIs: googleapis client uses OAuth tokens only
  • API Key Management: Keys stored in secure environment variables, never exposed to client

External Service Data Handling

Service Data Sent Risk Mitigation
Groq API Resume + JD (for improvement) Encrypted HTTPS. Groq doesn't train on user data. Delete after processing.
Tavily API Company name + role No personal data. Only company research. Public information only.
Google APIs OAuth token + file IDs Google's data handling. User's account. OAuth scoped to Drive/Sheets only.

Compliance & Standards

  • ✅ GDPR Compliant: No data retention, user controls everything via Google Drive
  • ✅ No Tracking: No analytics on resume content or user behavior
  • ✅ Open Source Friendly: Can be self-hosted for complete privacy control

Deployment Steps

  1. Connect GitHub repo to Vercel:
    • Go to vercel.com/new → Import GitHub repository
    • Vercel auto-detects Next.js — no config needed
  2. Configure OAuth: Add your Vercel production URL to Google OAuth authorized redirects in Google Cloud Console
  3. Set Environment Variables in Vercel Dashboard:
    • GOOGLE_CLIENT_ID — OAuth client ID
    • GOOGLE_CLIENT_SECRET — OAuth client secret
    • NEXTAUTH_SECRET — NextAuth session encryption key
    • NEXTAUTH_URL — Vercel production URL (e.g. https://your-app.vercel.app)
    • GROQ_API_KEY — Groq LLM API key
    • TAVILY_API_KEY — Tavily search API key
  4. Deploy via Git (recommended):
    # Push to main → Vercel auto-deploys production git push origin main # Or use Vercel CLI npx vercel --prod
  5. Verify Deployment: Vercel provides deployment URL immediately. Check /api/tailor endpoint health.
